Introduction

Another consensus algorithm

Why raft?

  • The problem with Paxos
    • Difficult to understand
    • Architecture not suited for most practical systems
    • Real implementation differs from Paxos → build on top unproven system
  • The goal
    • Understandable consensus algorithm

Particularities

  • Decomposition
    • Leader election
    • Logs replication
    • Safety
  • State space reduction
    • Reduce non-determinism
    • Reducde the way server are consistents
  • Use of novel features
    • Stronger leader
    • Leader election → randomized timers
    • Membership changes → joint consensus

Replicated state machines

  • Safety → never return a wrong result
  • Non-byzantin
  • Fail-stop
